Colleagues receive shapefiles per floor from CAD and GIS tools and needed a way to turn them into valid IMDF archives without installing Python or learning geospatial libraries, so this runs as a full-stack web app — a FastAPI + GeoPandas/Shapely backend behind a React + TypeScript frontend — hosted on a shared Windows PC that anyone on the team can reach from a browser.

The wizard auto-detects IMDF feature types straight from shapefile filenames, then walks through level, category, and unit-mapping configuration, including a lookup for company-specific unit category codes. From there it auto-generates footprint, building, and venue features directly from unit geometry, so those don't have to be modeled by hand.

Review happens on an interactive MapLibre map paired with a feature table — geometry is colored by category, selection is bidirectional between map and table, and keyboard shortcuts (<code>Ctrl+Z</code>, <code>Escape</code>, <code>Enter</code>) keep the review pass fast. Validation — geometry quality, spatial containment, opening placement — runs inline and shows up as filterable rows next to the real features, not as a separate error report, before the final IMDF archive gets exported.
